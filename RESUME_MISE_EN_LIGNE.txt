❓ QUESTIONS FREQUENTES:


Q: Pourquoi Netlify + Supabase?
A: Combinaison parfaite: déploiement auto + base PostgreSQL managée + gratuit
   Alternative: Vercel + Prisma, Railway, Heroku (mais plus compliqué)


Q: Et si je veux continuer avec Express localement?
A: C'est possible! Les deux modes coexistent:
   - Développement: npm run dev:local (Express + PostgreSQL Docker)
   - Production: Git push → Netlify Functions + Supabase


Q: Les coûts?
A: Gratuit jusqu'à ~1M requêtes/mois. Après: $0.50 par million requête.
   Supabase: Gratuit jusqu'à 500MB storage, $25/mois après.


Q: Comment je reviens en arrière?
A: Vous gardez votre code Express! Vous pouvez redéployer n'importe où.
   Netlify/Supabase ne sont que des hébergeurs.


Q: Comment je update l'app après le deploy?
A: 1. Modifiez le code localement
   2. git push origin main
   3. Netlify redeploy auto
   4. ✅ Nouvelle version live
